# TrustMed-AI — Drug Knowledge Assistant

**Tech Stack:** Rasa (DIET), Neo4j, Knowledge Graphs, Python, Docker, REST APIs

⚠️ **Medical Disclaimer:**  
TrustMed-AI is a technical demonstration for information retrieval and graph exploration. It is not a medical device and does not provide medical advice. Clinical decisions should always be made by licensed professionals.

## Overview
TrustMed-AI is an intelligent conversational assistant designed to answer complex drug and pharmaceutical queries using a structured medical knowledge graph. Instead of relying on ungrounded language model generation, the system retrieves answers through explicit graph traversal over authoritative medical concepts, making responses transparent, explainable, and traceable.

The assistant supports natural-language questions such as:
- “What are the active ingredients in *Zestril*?”
- “List products that contain *ibuprofen*.”
- “What’s the generic for *Zestril*?”
- “Find substitutes with the same active ingredients.”
- “Common co-ingredients with *ibuprofen*.”

## Motivation
Medical question-answering systems face two major challenges:
1. **Hallucination risk** when relying on unstructured text or purely generative models.
2. **Lack of explainability**, especially when answers are derived from opaque embeddings.

TrustMed-AI addresses these issues by grounding all responses in a structured knowledge graph derived from standardized medical ontologies, ensuring answers are traceable to explicit relationships between drugs, ingredients, and formulations.

## System Architecture
TrustMed-AI follows a modular, end-to-end architecture:

- **Natural Language Understanding:**  
  Rasa NLU (DIETClassifier) handles intent classification and entity extraction (e.g., product names, ingredients, brands, numeric constraints).
- **Action Server Layer:**  
  A custom Python action server acts as a façade, translating user intents into structured knowledge queries.
- **Knowledge Graph Backend:**  
  Neo4j stores drug concepts and relationships derived from UMLS (RxNorm + SNOMED CT), enabling fast and expressive graph traversal.
- **Interfaces:**  
  The system is exposed via REST APIs and a lightweight web chat UI.
- **Deployment:**  
  All components are orchestrated using Docker Compose for reproducibility.

This separation allows the NLU, reasoning logic, and data layer to evolve independently.

## Knowledge Graph & Data
The core of TrustMed-AI is a large-scale medical knowledge graph built from the **UMLS Metathesaurus**:

- **Scale:** ~354,000 medical concepts and over **23 million relationships**
- **Sources:** RxNorm and SNOMED CT
- **Representation:**  
  Concepts are modeled as nodes, with typed relationships capturing brand–generic links, active ingredients, formulations, and hierarchical relations.

To support rapid iteration and evaluation, the system also includes a **focused overlay dataset** (e.g., ACE inhibitors + common combinations), enabling safe testing before querying the full graph.

## Trust & Transparency
Each response generated by TrustMed-AI includes a **Trust Score (0–100)** along with a short explanation of why the score was assigned. The score combines:
- Data reliability (source coverage),
- Query quality,
- NLU confidence.

When available, external evidence identifiers (e.g., openFDA label IDs) are attached to responses. This design makes the system’s behavior transparent and reviewable, which is critical in medical contexts.

## Key Capabilities
- Brand ↔ generic resolution
- Ingredient and co-ingredient discovery
- Substitute identification based on active sets
- Strength extraction from product names
- Relationship-aware exploration beyond keyword matching

All answers are produced via deterministic graph queries rather than free-form generation.

## Learnings
- Knowledge-graph grounding significantly reduces hallucination risk compared to text-only RAG approaches.
- Separating NLU from reasoning logic simplifies debugging and extensibility.
- Even large ontologies benefit from curated “overlay” datasets for evaluation and iteration.
- Trust signaling (scores + evidence) improves system interpretability and user confidence.

## Limitations & Future Work
- Extend trust scoring with learned confidence calibration.
- Integrate vector-based retrieval for hybrid KG + embedding workflows.
- Expand evaluation beyond qualitative testing to user-centric metrics.
- Explore domain adaptation to additional therapeutic areas.
